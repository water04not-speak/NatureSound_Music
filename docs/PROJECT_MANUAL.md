# 万物之声项目说明书

## 一、项目概述

“万物之声 NatureSound Music”是一款基于 HarmonyOS ArkTS 和 ArkUI 开发的音乐播放与非遗音乐内容展示应用。项目以移动端音乐 App 为基础形态，围绕播放、歌单、歌词、收藏、搜索、非遗音乐、互动社区、用户中心和主题换肤等模块组织功能。

本项目适合作为 HarmonyOS 课程设计、作品集展示和代码讲解项目。公开仓库保留完整工程结构、核心代码和演示数据结构，但不包含受版权保护的完整音频、歌词、专辑封面、歌手图片或第三方商业素材。

## 四、核心功能

### 1. 音乐播放

播放器核心封装在 `entry/src/main/ets/utils/AvPlayerBridge.ets` 中。页面不直接操作底层播放器，而是通过播放管理器调用播放、暂停、切歌、上一首、下一首、进度跳转等方法。

播放器切歌时会同步当前歌曲的封面、歌名、歌手、播放地址、收藏状态和歌词信息，并在歌曲切换前清空旧歌词状态，避免歌词界面显示上一首歌的内容。

### 2. 歌词同步

歌词页位于 `entry/src/main/ets/pages/Lyric.ets`。歌词来源支持三种形式：

| 类型 | 说明 |
| --- | --- |
| `lyricLines` | 已解析好的带时间戳歌词行 |
| `lyric` | LRC 文本，运行时解析成时间戳歌词 |
| `lyrics` | 普通文本数组，根据歌曲时长做平均时间打点 |

当前实现优先根据当前播放歌曲 `id` 匹配歌词，找不到时才按播放列表索引兜底。切歌后会重置歌词缓存、当前高亮行和滚动进度，保证歌词和音乐对应。

### 3. 数据持久化

`MusicDatabase.ets` 负责创建 RDB 数据库和歌曲表，`MusicRepository.ets` 负责封装业务访问。主要能力包括：

- 初始化公开版演示歌曲数据。
- 读取全部歌曲。
- 按关键词搜索歌曲。
- 收藏和取消收藏。
- 按歌曲编号读取最新歌词和收藏状态。
- 支持本地私有资源覆盖公开版演示数据。

### 4. 后台播放

`AvSessionManager.ets` 负责 AVSession 能力，主要用于同步后台播放状态、媒体元数据和后台任务。这样用户离开前台页面后，系统媒体控制区域仍能展示当前歌曲信息并控制播放状态。

### 5. 非遗音乐模块

非遗音乐数据位于 `HeritageMusicData.ets`，页面通过地域和音乐类型组织内容。公开版使用演示数据结构和占位资源，重点展示非遗主题内容组织、列表展示和页面跳转能力。

### 6. 本地私有资源

公开仓库不会提交真实音乐资源。若需要在本机演示完整播放效果，可以在以下忽略文件中配置自己已获得合法授权的测试资源：

```text
entry/src/main/resources/rawfile/local_music.private.json
```

该文件已被 `.gitignore` 忽略，不会随代码提交到 GitHub。建议只在本机或已确认授权的演示环境使用。

## 五、运行方式

1. 使用 DevEco Studio 打开项目根目录。
2. 等待工程同步完成。
3. 配置 HarmonyOS 调试签名。
4. 选择模拟器或真机运行 `entry` 模块。
5. 若只使用公开仓库数据，项目会展示占位音乐结构和页面能力。
6. 若需要本地完整播放效果，请准备合法授权的音频、歌词和封面资源，并写入 `local_music.private.json`。

## 六、版权与公开发布策略

为降低版权风险，公开仓库遵循以下规则：

- 不提交完整真实音频文件。
- 不提交完整真实歌词文本。
- 不提交第三方专辑封面、歌手图片或商业素材。
- 不提交下载缓存和临时文件。
- 公开版仅保留演示数据结构、占位歌词和项目截图。
- 私有音乐资源只放在被忽略的本地配置文件中。

## 七、代码讲解建议

讲解时可以按以下顺序展开：

1. 先介绍项目主题：音乐播放器加非遗音乐内容展示。
2. 再介绍工程分层：页面、模型、工具、资源、文档。
3. 重点讲 `AvPlayerBridge`：如何封装 AVPlayer 并维护播放状态。
4. 重点讲 `Lyric`：如何解析 LRC、按播放时间高亮歌词、避免歌词错配。
5. 讲 `MusicRepository`：如何通过 RDB 管理歌曲、收藏和搜索。
6. 讲 `AvSessionManager`：如何支持后台播放状态同步。
7. 最后说明公开仓库的版权安全策略和本地私有资源配置方式。
